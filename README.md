# 电子贺卡系统(第一阶段:卖家自营版)

扫码打开的 H5 电子贺卡。3 套主题 × 8 套主题色 × 原创卡通动物,纯静态站,零后端、零数据库。

## 文件说明

| 文件 | 作用 | 给谁看 |
|---|---|---|
| `card.html` | 贺卡展示页,买家扫码打开的就是它 | 买家 |
| `make.html` | 生成器,填信息→出 JSON+二维码 | **只有你自己** |
| `index.html` | 根路径跳转到演示卡 | 任何人 |
| `data/编号.json` | 每张贺卡一个配置文件 | — |
| `data/编号/1.jpg…` | 每张贺卡的照片 | — |

演示卡:部署后访问 `/card.html?id=demo`

## 部署(免备案测试,二选一,都免费)

### 方案 A:Cloudflare Pages(推荐,国内可达性相对好)
1. 把本文件夹推到 GitHub 仓库(可设为 private)
2. dash.cloudflare.com → Workers & Pages → Create → Pages → 连接该仓库
3. 构建设置全部留空(纯静态),Deploy
4. 得到 `https://xxx.pages.dev`,填进生成器的"站点域名"

### 方案 B:Vercel
1. vercel.com → Add New Project → 导入 GitHub 仓库 → Deploy
2. 得到 `https://xxx.vercel.app`

> ⚠️ 免备案域名在微信内可能偶发"非微信官方网页"提示或被拦截,属正常现象。
> 正式运营请并行提交国内域名备案,备案后把站点迁到国内服务器/OSS,生成器里换个域名即可,贺卡数据不用动。

## 接单流程(每单约 3 分钟)

1. 本地或线上打开 `make.html`
2. 选主题、主题色,填买家提供的昵称/祝福语,上传照片(自动压缩)
3. 点"生成贺卡" → 下载 JSON、压缩后照片、二维码图
4. `编号.json` 放进 `data/`,照片放进 `data/编号/`
5. `git add . && git commit -m "order" && git push` → 平台自动重新部署(约 1 分钟)
6. 打开链接自己先验一遍,没问题就把**二维码图**发给买家

## 有效期机制

- 生成时填天数(默认 30 天),写进 JSON 的 `expireAt`
- 过期后买家扫码看到"已过期,请联系赠送人"——这是你的**续费/复购入口**
- 续期 = 改 JSON 里的 `expireAt` 再 push,1 分钟搞定

## 防白嫖与安全

- `make.html` 不要出现在任何对外链接里;上架商品图只放贺卡效果图
- 担心被人翻到生成器:把 `make.html` 改个随机文件名,如 `mk-x7p2q.html`
- 贺卡编号是 7 位随机串,无法被遍历猜出

## 背景音乐

生成器里直接选:

- 内置 3 首(`assets/bgm/`):生日快乐歌、原创浪漫旋律、小星星。前后两首词曲为公有领域,音频为本项目原创合成演奏,可商用
- 自定义上传 MP3(≤5MB):生成后随照片一起下载,放进 `data/编号/`。**勿用流行歌曲音频,有版权风险**
- 买家打开贺卡后右上角有音乐按钮;微信内点"轻触开启"会尝试自动播放
